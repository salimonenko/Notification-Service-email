Система эмулирует массовую отправку сообщений e-mail.
Система рассылает сообщения в виде двух очередей: обычной и срочной. Для реализации работы этих очередей есть асинхронный брокер сообщений. Сообщения фактически НЕ отправляются, вместо функций типа mail() используется функция-заглушка. 
Изначально обычные сообщения, после их "отправки", помещаются в базу данных со статусом "0". После "удачной отправки" сообщений им присваивается статус "3" (в функции-заглушке). Если сделать контроль факта отправки сообщений, можно добавить и другие статусы, тогда в БД будут содержаться именно они. 

Сообщения (реализовано только для обычных, несрочных сообщений) отправляются заданными порциями. Для каждой порции делается один SQL-запрос для 

Рассылка сообщений, а также изменение их статуса в базе данных, производится в цикле, по определенному количеству (заданное число) сообщений за один SQL-запрос. В рамках которого производится сохранение в БД истории отправленных сообщений и их статуса оправки.

Реализован простой вариант политики Exactly-once: сообщение отправляется до тех пор, пока приемник не подтвердит получение; но, не более заданного максимального числа попыток. Недостаток реализации: соответствующая очередь отправки сообщений приостанавливается на определенное заданное время (таймаут) ожидания.
Для этой цели реализован повтор отправки сообщений в течение заданного числа попыток.
